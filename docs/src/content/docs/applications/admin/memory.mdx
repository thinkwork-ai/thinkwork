The Memory page is the operator view for retained Hindsight records. It reports active user and Space memory, shows bank and scope evidence, and lets operators inspect what was retained before agents retrieve it through Context Engine.

**Route:** `/settings/memory`
**Route wrapper:** `apps/web/src/routes/_authed/settings.memory.tsx`
**Page implementation:** `apps/web/src/components/settings/SettingsMemory.tsx`

The Knowledge Graph subview remains available at `/settings/memory/knowledge-graph` when ThinkWork Brain graph infrastructure or legacy Cognee evidence is present.

## Layout

The page has three primary controls and one operator status band:

- **Memory mode** — shows the active memory engine. Hindsight appears as the core memory engine; ThinkWork Brain/Cognee graph infrastructure appears only as operator diagnostic evidence.

- **View toggle** — table / graph inspection for retained memory evidence.
- **Search bar** — text search across operator-visible Hindsight memory records, backed by `memoryRecords(scope: OPERATOR)`.
- **Refresh** — manual re-fetch.

Below the controls, the page renders either a `DataTable` of memory records or an interactive 3D graph.

## Memory Records

The table view shows:

| Column  | Notes                                                              |
| ------- | ------------------------------------------------------------------ |
| Created | When the record was created                                        |
| Updated | When the record was last updated                                   |
| Bank    | Hindsight bank id, such as a user or Space bank                    |
| Scope   | Owner type/id when derivable                                       |
| Type    | Strategy badge: semantic, summary, preference, episode, reflection |
| Memory  | Truncated content preview                                          |

Clicking a row opens a detail sheet with full content, bank/scope metadata, and retention metadata. Operator-wide rows are read-only in this pass; cross-bank deletion is intentionally disabled until a safer scoped mutation exists.

- **Hindsight memory** — canonical user and Space memory for this pass.
- **AgentCore managed memory** — an explicit low-cost/development opt-out; records are immutable by default.
- **ThinkWork Brain/Cognee graph infrastructure** — optional ontology/graph/warehouse evidence, not the user or Space memory provider.

## Graph View

When graph infrastructure is enabled, the graph view renders entities and relationships with `MemoryGraph` (React Force Graph 3D, three.js, and d3-force). Treat it as ThinkWork Brain graph diagnostics, not proof that user or Space memory is backed by Cognee.

<Aside type="note">
  The graph view is a Brain infrastructure inspection tool. It is not the full
  ThinkWork Brain projection: distillation, ontology processing, wiki pages,
  aliases, and source citations are follow-up work.
</Aside>

## Workflows

### Audit What an Agent Remembers

1. Open `/settings/memory`.
2. Search for a term, bank id, owner id, or record text.
3. Click a row to read the full content and bank/scope metadata.
4. Use the bank and scope evidence to verify whether the record belongs to a user or Space.

### Debug Memory Results

1. Run the query from the Context Engine tool in `/settings/tools`.
2. If a memory hit looks wrong, open `/settings/memory`.
3. Search for the same term, bank id, or owner id.
4. Inspect the raw record and backend metadata.

### Delete Sensitive Memory

1. Search for the sensitive term.
2. Click into each matching record.
3. Use the bank/scope metadata to prepare a reviewed backend redaction.

Cross-bank delete is disabled in the operator table for this pass. For bulk redaction, operate at the backend layer with a reviewed script.

## Data Model

- **Hindsight user memory** — Hindsight memory keyed to the requester.
- **Hindsight Space memory** — Hindsight memory keyed to the current Space. It stays with the Space and is available to authorized members and agents.
- **AgentCore managed memory** — Aurora `memory_records` table with strategy, namespace, content, tags, confidence, access count, and bi-temporal dates.
- **ThinkWork Brain graph diagnostics** — optional graph infrastructure evidence surfaced when the Brain substrate is installed or legacy Cognee is available.
- **Namespaces** — records are namespaced by strategy and agent; prefixes such as `semantic_`, `preferences_`, and `episodes_` identify the retention strategy.

## Known Limits

- **Graph diagnostics are not memory authority.** Hindsight is the user and Space memory path for this pass.
- **ThinkWork Brain distillation is deferred.** This page proves retained-memory inspection; it does not prove Brain distillation.
- **Wiki projection is separate.** Wiki pages are compiled context, not an upstream source of truth for retained memory.
- **Operator rows are read-only.** Editing/deleting across Hindsight banks is intentionally disabled in this pass.
- **Graph view lacks drill-in.** Clicking a node does not open a detail sheet.
- **Search is semantic-only.** It can miss exact IDs or unusual strings.
- **No bulk edit or bulk delete.** Every mutation is one record at a time.
