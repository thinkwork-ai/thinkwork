Workspace is the default Space Studio tab. It edits the files that apply only inside the current Space: procedures, customer context, project notes, team norms, and workflow instructions that should not become tenant-wide defaults.

**Route:** `/spaces/:spaceId/workspace`

Opening `/spaces/:spaceId` redirects here.

For authoring guidance, start with [Build a Space](/guides/spaces/build-a-space/). This page documents the Workspace tab itself.

## What to put in a Space workspace

Use Space workspace files for local context:

- customer, project, or team notes;
- inbox or channel procedures;
- workflow-specific runbooks;
- escalation rules for this workroom;
- context the tenant platform agent should read only when work happens in this Space.

Use [Tenant Agent](/applications/admin/agents/) for baseline files that apply across the tenant. Use folder specialists for reusable delegated behavior.

<Aside type="note">
  A Space workspace does not clone the tenant platform agent. It adds local
  workroom context for turns that happen in the Space. During a turn, the active
  Space mounts as singular `Space/` in `/workspace`.
</Aside>
