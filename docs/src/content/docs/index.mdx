## Six core concepts, one deploy

<CardGrid>
  <Card title="Agents" icon="rocket">
    Managed agents powered by Bedrock, connected agents that bring their own runtime. Agent Templates for fleet-wide configuration. Skill packs for reusable capabilities.
  </Card>
  <Card title="Threads" icon="document">
    The universal work container. Every conversation, task, automation, and email lands in a thread with full history, context, and audit trail.
  </Card>
  <Card title="Connectors" icon="puzzle">
    Bring in external channels, events, and tool systems. Slack, GitHub, Google Workspace, plus MCP-based tool connectors and custom recipes.
  </Card>
  <Card title="Automations" icon="setting">
    Scheduled jobs, event-driven triggers, and routines that run on your schedule. Step Functions under the hood.
  </Card>
  <Card title="Control" icon="approve-check">
    Guardrails, budgets, and usage tracking. Know what your agents are doing, how much they cost, and where the boundaries are.
  </Card>
  <Card title="Knowledge" icon="open-book">
    Document-backed RAG via Bedrock Knowledge Bases. Upload documents, agents retrieve what they need inline.
  </Card>
</CardGrid>

## Quick start

```bash
npm install -g @thinkwork/cli
thinkwork login
thinkwork init -s dev
thinkwork deploy -s dev
thinkwork doctor -s dev
```

That's it. Five commands, one AWS account, production-grade agent infrastructure.
