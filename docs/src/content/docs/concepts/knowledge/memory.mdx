Long-term memory is how ThinkWork carries information forward from previous work. Unlike document knowledge, which starts with uploaded source files, long-term memory starts with what happened in threads.

## What long-term memory means in ThinkWork

Long-term memory helps agents retain useful facts across turns or across related work, such as:

- User or team preferences
- Ongoing project context
- Prior decisions and follow-up items
- Summaries synthesized from earlier conversations

ThinkWork ships with **AgentCore managed memory always on** and **Hindsight as an optional add-on** that runs alongside it.

## AgentCore managed memory — always on

Every ThinkWork deployment provisions an AWS Bedrock AgentCore Memory resource as part of the `thinkwork` Terraform module. The agent container automatically emits a `CreateEvent` for every turn (user message + assistant response), and AgentCore's background strategies extract facts into four namespaces:

- **Semantic** — similarity-based recall of cross-thread facts
- **Summarization** — synthesized per-session carry-forward context
- **User-preference** — durable preference capture
- **Episodic** — remembered events and prior interactions

The agent reads them back via the `recall()` tool. There is no need for the model to call `remember()` for routine facts — it only exists for user-driven "please remember X" requests. Retention is automatic and requires zero prompting.

## Hindsight — optional add-on

**Hindsight** is an additional long-term memory system that runs alongside managed memory, not instead of it. It runs on ECS Fargate behind an ALB and adds a fuller explicit memory lifecycle with entity-graph traversal and cross-encoder reranking:

- **Retain** stores candidate memories from conversations
- **Recall** retrieves relevant memories with entity-graph and BM25 signals
- **Reflect** periodically synthesizes and compresses memory over time

Enable it with `enable_hindsight = true` in your deployment config. When enabled, the agent gets both tool sets — managed `remember()`/`recall()`/`forget()` AND `hindsight_retain`/`hindsight_recall`/`hindsight_reflect`. The `remember()` tool dual-writes to both backends so explicit memories land in both systems.

<Aside type="tip">
Managed memory gives you automatic retention out of the box. Only enable Hindsight if you need its entity-graph retrieval, cross-encoder reranking, or batch reflection features — they're not a replacement for managed memory.
</Aside>

## Long-term memory versus thread history

Thread history is the complete record. Long-term memory is the selective carry-forward layer.

- **Thread history** preserves everything in the thread database
- **Long-term memory** surfaces only the pieces worth reusing later

That distinction matters because long threads are still auditable even when only a smaller set of facts is recalled into the next turn.

## Long-term memory versus document knowledge

Use long-term memory for information learned from work. Use document knowledge for information loaded from source material.

In practice, many agents use both:

- document knowledge grounds answers in docs and policies
- long-term memory carries forward context from prior interactions
