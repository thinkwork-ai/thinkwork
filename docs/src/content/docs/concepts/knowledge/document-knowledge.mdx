Document knowledge is Thinkwork's current document retrieval layer. You upload source material, Thinkwork indexes it into a Bedrock Knowledge Base, and managed agents can retrieve relevant passages during a turn.

## What it is good at

Use document knowledge for:

- Product docs and internal runbooks
- Policies and operating procedures
- FAQ-style support content
- Long-form reference material that changes occasionally

It is a grounding layer, not a replacement for memory. Documents answer, "what does the organization know?" Memory answers, "what has this agent or team learned across interactions?"

## Storage model

```
Document upload
    → S3 knowledge bucket
    → Bedrock Knowledge Base sync
    → embeddings + vector index
    → retrieval during agent turns
```

The source files stay in S3 in your AWS account. Retrieval uses Bedrock Knowledge Bases backed by OpenSearch Serverless.

## Operational notes

- Documents do not automatically resync when source files change
- Chunking strategy is set when the knowledge base is created
- OpenSearch Serverless sets a meaningful cost floor, even at low usage

<Aside type="caution">
If you are cost-sensitive, decide whether you need a full Bedrock Knowledge Base before enabling it broadly. The retrieval experience is strong, but the OpenSearch Serverless minimum cost is material for small deployments.
</Aside>
