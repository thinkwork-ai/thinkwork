Memory is the umbrella for the context Thinkwork agents use beyond the latest message. In practice, that includes both retrieved source material and retained context from prior work.

This section explains how Thinkwork combines document knowledge, long-term memory, retrieval and context assembly, plus where the knowledge graph direction may go next.

## What belongs under Memory

Thinkwork separates three related but different ideas:

- **Threads** keep the full record of work
- **Memory** decides what context gets surfaced into a turn
- **Agents** use that context to act

That framing keeps the docs cleaner and avoids making the platform feel like document RAG is the whole story.

## Current reality

In the current open source docs, the most concrete implemented memory story is:

1. document retrieval through Bedrock Knowledge Bases
2. AWS AgentCore LongTerm memory as the default long-term memory system
3. Hindsight as an optional alternative memory engine
4. context assembly inside the agent turn

The default long-term memory setup includes four built-in strategies:

- semantic
- summarization
- user-preference
- episodic

That is enough to build useful internal agents today without implying a larger memory platform than the repo currently documents.
